Linux internals · Containers · Python

Toy Container Runtime

View repository ↗

A minimal container runtime built to explore how containers work below Docker and Podman. The project examines process isolation, Linux namespaces, filesystem boundaries, mounts, and container process execution.

Why I built it

I wanted to move beyond using container abstractions and understand the kernel primitives that container runtimes assemble.

Concepts demonstrated

  • Linux namespaces
  • Process isolation
  • Root filesystem handling
  • Mount and process lifecycle management

Kubernetes · Ansible · Air-gapped systems

RKE2 Setup

View repository ↗

An Ansible-driven workflow for deploying RKE2 in a disconnected environment. The repository focuses on reproducible installation, offline assets, node configuration, and reducing manual cluster setup.

Why I built it

Air-gapped environments introduce additional operational constraints. I wanted the cluster installation to remain understandable, repeatable, and maintainable.

Concepts demonstrated
